MC6 Keyboard only UNCHECKED

Check keyboard accessibility

  • Tab to all links, buttons and form controls. If you can’t tab to these controls then the Tab Indexes may be incorrect or missing.
  • You can not tab to non-functional elements (such as headings, paragraphs or labels). Labels that are focusable elements themselves are misleading and don’t serve the purpose they were designed for. Any labels that can be focused upon please raise as a defect.
  • The focus of the fields are clear. This will either be with a red or yellow outline if using the nhs toolkit. Other focus colours will have to match WCAG colour contrast requirements.
  • The focus does not move or select hidden elements. An instance of this would be where the focus seems to disappear then reappear when tabbing or moving on. This would indicate that hidden fields gained the focus attention.
  • The focus does not get trapped in any element and does not move.
  • Scripted events (for keyboard, mouse and touch) only occur when expected, for example events that would happen when a button is clicked must also be available if keyboard navigation is used (focus is on the element then space or return key is pressed).
  • The visual order for tabbing progress is left-to-right and top-to-bottom and forms a logical pattern. If the tab order is down lists or in a random sequence then these are failures.
MC7 Zoom & Resizing Unchecked

Test that text sizes and zooms correctly

  • Size your browser window to 1280px wide. (You can use the web developer toolbar in Firefox or Chrome.)
  • Zoom into 400%. All content including tables should be focusable and no scroll bars should appear.

  • Whilst zoomed in at 400%: Try navigating with keyboard only, if this causes issues then these need to be raised as defects.

MC9 Links UNCHECKED

Meaningful links

  • Check all links and link text are clear and accurate to their purpose and role.
  • Check all links for source / destination duplication. If the same link appears ensure it directs to the same source. For example, 2 Contact us links on the same page should go to same source. Different link names are required if they are to be different link destinations.
  • If multiple links exist please question why links have been duplicated. Duplication can cause confusion to users.
  • Check for duplication of links like ‘Change’ on Check Your Answers pages. All these links should be unique and describe what each change link is for. For example, Change forename, change surname.
  • Check for links that download files. Any link that will open, download or view a file must have the file type specified in the link. We don’t have to include the size of the file but this may be useful as well.
  • Check any links that open content in a new tab or window clearly states this next to the link. Check in assistive technologies that this also works correctly.

MC13 Tab indexes UNCHECKED

Check that focus is handled in sequential order by keyboard navigation

  • Check tabindex code is not included on fields or elements.
    This way the elements on the screen will gain focus based on the logical structure of keyboard navigation on the screen. (left to right then top to bottom).
  • Check tab indexes do not have negative values (usually tabindex="-1") unless there is a valid reason for doing so.
    By using negative tab indexes this This means that the element is not reachable via sequential keyboard navigation.
    The only exemption to this case is for error summaries or links. These are set specifically with -1 to gain immediate focus when an error occurs.

MC15 Time outs UNCHECKED

Give users time to read and act

  • If you have a time out setting applied then ensure both visual and non visual users need to be made aware of the timeout period coming to an end.
  • This is in the form of a prompt that will include a timer which will count down the remainder of the timeout session. This is, for example five mins before the end of a 20 minute timeout.
  • You also need to give the user the chance to interact with the webpage to prevent the timeout from happening or be able to extend their session.
  • If the user selects to extend the session or makes an action to stop the time out then the original webpage they were on should appear. The webpage should not be refreshed or changed in any way from what it showed before the timeout.
